# คู่มือการใช้งาน System Log

## สารบัญ
1. ข้อมูลเบื้องต้น
2. รายละเอียดฟีเจอร์
3. วิธีการใช้งาน
   - ดูบันทึก System Log
   - ค้นหาและกรองข้อมูล Log
4. รายละเอียดข้อผิดพลาด
   
## ข้อมูลเบื้องต้น
System Log เป็นระบบที่ใช้ในการบันทึกและแสดงประวัติการทำงานของระบบ เช่น ข้อผิดพลาด การเข้าสู่ระบบ และเหตุการณ์ที่เกี่ยวข้องกับระบบ ช่วยให้ผู้ดูแลระบบสามารถติดตามปัญหาและวิเคราะห์ข้อมูลได้ง่ายขึ้น

เป้าหมายของระบบ System Log 
- ช่วยให้ผู้ดูแลระบบสามารถตรวจสอบและติดตามปัญหาที่เกิดขึ้น  
- ช่วยให้ผู้ใช้สามารถดูประวัติของกิจกรรมที่เกิดขึ้น  
- ป้องกันข้อผิดพลาดในการใช้งานระบบโดยสามารถตรวจสอบย้อนหลังได้  

## รายละเอียดฟีเจอร์
✔ แสดงรายการบันทึกกิจกรรมของระบบ (System Events)  
✔ ค้นหาและกรองบันทึกตามประเภทหรือผู้ใช้งาน  
✔ แสดงรายละเอียดข้อผิดพลาดเพื่อช่วยในการแก้ไข  
✔ เก็บข้อมูลเวลาที่เกิดเหตุการณ์และรายละเอียดของกิจกรรม  

## วิธีการใช้งาน

### เข้าสู่ระบบ
1. เปิดเบราว์เซอร์และเข้าไปที่ URL ของระบบ (เช่น `http://127.0.0.1:8000/login`)  
2. กรอก ชื่อผู้ใช้ (Username) และ รหัสผ่าน (Password)  
3. กดปุ่ม Login เพื่อเข้าสู่ระบบ
   
### ดูบันทึก System Log
1. ไปที่เมนู System Log บนแถบเมนูด้านซ้าย  
2. ระบบจะแสดงบันทึกที่เกิดขึ้นตามลำดับเวลา  
3. สามารถดูรายละเอียดของแต่ละรายการได้ในคอลัมน์ Date, Time, User, Event, Type และ Description

### ค้นหาและกรองข้อมูล Log
1. การค้นหา
   - ใช้ช่อง Search เพื่อค้นหาข้อมูล เช่น `LOCAL.ERROR` หรือ `UserID`
2. การกรองข้อมูล  
   - กดปุ่ม Filter และเลือกเงื่อนไขที่ต้องการ เช่น:  
     - ช่วงเวลา (เลือกวันที่เริ่มต้น - สิ้นสุด)  
     - ผู้ใช้ (เลือกดูเฉพาะผู้ใช้ที่ต้องการ)  
     - ประเภทของ Log (`ERROR`, `WARNING`, `INFO`)

## รายละเอียดข้อผิดพลาด
| ประเภทข้อผิดพลาด | คำอธิบาย | สาเหตุที่เป็นไปได้ |
|----------------|------------|----------------|
| `LOCAL.ERROR` | ข้อผิดพลาดทั่วไปของ Laravel | คำสั่งผิดพลาด หรือไฟล์ที่หายไป |
| `Method Illuminate\Support\Collection::paginate does not exist` | ฟังก์ชัน `paginate()` ใช้ไม่ได้ | ใช้ผิดที่ หรือข้อมูลผิดประเภท |
| `Undefined array key "status"` | ไม่มีตัวแปร `status` ในหน้า View | ตัวแปรไม่ได้ถูกส่งไปยัง Blade Template |
| `htmlspecialchars(): Argument #1 ($string) must be of type string, array given` | ข้อมูลไม่อยู่ในรูปแบบ String | ส่งข้อมูล Array ไปแทนที่ควรเป็น String |
| `SQLSTATE[42S02]: Base table or view not found` | ตารางฐานข้อมูลไม่พบ | ยังไม่ได้รัน Migration หรือชื่อตารางผิด |
